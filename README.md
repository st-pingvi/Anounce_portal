# Портал анонсов «Московское долголетие»

Репозиторий с материалами проекта портала анонсов на базе Outline. Внутри собраны:

- контентная и информационная архитектура портала;
- deployment-пакеты для self-hosted Outline;
- branded portal kit по ТЗ;
- локальный статический прототип интерфейса.

## Что входит в публикацию

- `outline-portal-kit/` — основной проектный kit: архитектура, шаблоны, схема Airtable, автоматизации и брендовые правила.
- `prototype-ui/` — локальный HTML/CSS/JS-прототип портала.
- `outline-deployment/` — production-ориентированный пакет деплоя Outline и связанной инфраструктуры.
- `deploy/outline/` — упрощённый пакет локального или серверного запуска Outline.
- `docs/`, `repository/`, `templates/` — вспомогательные материалы, справочники и шаблоны.

## Быстрый старт

### Локальный просмотр прототипа

Откройте файл:

`prototype-ui/index.html`

Прототип работает как статический интерфейс без отдельного backend-сервиса.

### Основной проектный kit

Стартовая точка:

`outline-portal-kit/README.md`

Там собраны:

- структура будущего Outline workspace;
- схема ролей и пользовательских сценариев;
- поля анонса и post-event блока;
- интеграции с Airtable, Zapier, Canva и календарями;
- брендовые ограничения и внедренческий чек-лист.

### Self-hosted Outline

Если нужен deployment:

- `outline-deployment/README.md` — основной production-вариант;
- `deploy/outline/README.md` — упрощённый пакет.

## Подготовка к публикации

В публичную версию не включаются:

- локальные `.env`-файлы и runtime-данные Outline;
- архивы экспорта и временные артефакты;
- побочные локальные эксперименты вне основного состава проекта.

Если планируете публиковать репозиторий на GitHub или GitLab, перед `push` проверьте:

1. что удалённый `origin` настроен;
2. что в Git не попали реальные секреты и серверные `.env`;
3. что README соответствует тому набору файлов, который вы действительно хотите показать внешней аудитории.

## Текущее состояние Git

Основные изменения проекта сохранены локально в ветке `main`.

На текущий момент удалённый `git remote` для репозитория не настроен, поэтому публикация в GitHub/GitLab ещё не выполнена.
