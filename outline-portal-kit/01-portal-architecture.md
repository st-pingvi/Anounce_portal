# Архитектура портала

## Контур решения

Портал строится вокруг Outline как пользовательского и редакторского слоя, но с сервисной обвязкой для требований ТЗ.

Базовый сценарий:

1. Инициатор создает анонс через форму.
2. Форма пишет запись в Airtable и фиксирует автора.
3. Zapier и сервисный backend валидируют обязательные поля и присваивают статус `Черновик`.
4. AI-слой генерирует короткий соцтекст до 200 символов.
5. Canva API или Python Pillow собирает брендированную афишу `1080x1920`.
6. Модератор подтверждает публикацию.
7. Документ публикуется в Outline.
8. Событие уходит в Google Calendar, VK, Telegram и экспортный контур `after55.moscow`.
9. После события инициатор дополняет карточку фото, метриками и ссылкой на отчет.

Принцип системы:

`Нет анонса — нет события`.

## Зоны ответственности

### Outline

- каталог событий;
- лендинги месяцев и подборок;
- внутренняя редакционная зона;
- страницы правил, шаблонов и инструкции;
- embed форм, календарей, Airtable views и внешних афиш.
- детальная карточка события;
- пост-ивент обновления через шаблоны и встроенные формы/ссылки.

### Airtable

- master table событий;
- статусы согласования;
- справочники площадок, районов, форматов, аудиторий;
- источники для Zapier-автоматизаций;
- лог публикации и версий.

### Zapier

- intake pipeline после формы;
- генерация AI-анонса;
- создание задач на Canva;
- публикация в Outline;
- обновление Google Calendar и уведомлений редакторам.
- публикация в VK и Telegram;
- экспорт в внешние каналы.

### Сервисный backend

- интеграция с SSO или mos.ru;
- модерация и бизнес-правила;
- FullCalendar/API слой для внешнего фронтенда при необходимости;
- Elasticsearch-индексация;
- экспорт в `after55.moscow`;
- пост-ивент загрузка файлов в S3/MinIO.

## Основные сущности

- `Event` — карточка события;
- `Venue` — площадка;
- `District` — район;
- `Program Line` — тематическая линия;
- `Poster Task` — задача на афишу;
- `Publication` — состояние публикации в Outline;
- `Calendar Export` — связь с Google Calendar.
- `Post Event Update` — пост-ивент данные;
- `Channel Delivery` — статус публикации в VK/Telegram/after55;
- `Approval` — запись модерации.

## Роли

- `Администратор портала` — управляет интеграциями, правами и шаблонами;
- `Редактор` — проверяет карточки, публикует и снимает с публикации;
- `Модератор` — утверждает или отклоняет событие;
- `Координатор площадки` — подает и обновляет события через форму;
- `Дизайнер/оператор афиш` — контролирует Canva-вывод;
- `Инициатор анонса` — сотрудники управленческой команды, сотрудники ЦМД, территориальные управляющие, менеджеры по партнерам, поставщики, лидеры клубов самоорганизации, кураторы молодых специалистов;
- `Читатель` — смотрит каталог, календарь и карточки.

## Что публикуется в Outline

- главная страница портала;
- "Афиша недели";
- "Афиша месяца";
- каталог мероприятий по темам;
- карточки конкретных событий;
- спецподборки: здоровье, творчество, цифровая грамотность, межпоколенческие события;
- служебные документы для редакции.

## Публичный и внутренний слои

Публичные коллекции:

- `Портал анонсов`
- `Каталог событий`
- `Афиша месяца`
- `Календарь`

Внутренние коллекции:

- `Редакция`
- `Шаблоны`
- `Операционные инструкции`

## Логика карточки события

Источник правды по полям — Airtable.

Источник отображения — документ Outline, созданный из шаблона.

Синхронизируемые поля:

- название;
- дата и время;
- площадка;
- район;
- короткий анонс до 200 символов;
- полное описание;
- формат;
- аудитория;
- ссылка на регистрацию;
- ссылка на маршрут;
- афиша;
- статус публикации.
- фото после события;
- посещаемость;
- отзывы;
- ссылка на отчет;
- статус `Завершено`.

## Форма создания анонса

Обязательные поля:

- дата и время начала;
- название;
- место проведения;
- формат;
- описание;
- для кого;
- кто внес.

Необязательные:

- ссылка на участие/регистрацию.

Автоматически:

- ФИО и роль инициатора;
- служебный ID записи;
- временные метки;
- статус модерации.

## Пост-ивент сценарий

После завершения события инициатор или редактор может:

- загрузить фото;
- заполнить посещаемость;
- добавить отзывы;
- прикрепить ссылку на отчет;
- перевести событие в статус `Завершено`.

## Нефункциональные требования в архитектуре

- responsive и PWA-режим для мобильного сценария;
- поиск с индексом Elasticsearch;
- время генерации целевого ответа менее 2 секунд для типового сценария;
- хранение файлов в S3/MinIO;
- SSO или mos.ru для авторизации;
- обязательная модерация перед публикацией.

## Почему такая схема

- Outline хорошо подходит для навигации, чтения, шаблонов и embeds;
- Airtable удобнее для табличной редакторской логики и статусов;
- Zapier закрывает интеграции без отдельного backend;
- Canva позволяет стандартизировать афиши без ручной верстки каждой карточки.
- backend нужен для SSO, индексации, внешних каналов и надежной модерации.
