# Спецификация формы подачи события

## Рекомендуемый вариант

Использовать `Airtable Form`, встроенный в Outline через `/embed`.

Альтернативный вариант:

- Tally form + Zapier + Airtable.

## Название формы

`Подать событие в портал анонсов`

## Структура формы

### Блок 1. Основная информация

- Название события
- Краткое описание
- Формат
- Тематическая линия
- Для кого событие

### Блок 2. Дата и место

- Дата
- Время начала
- Время окончания
- Площадка
- Район

### Блок 3. Организация

- Нужна ли регистрация
- Ссылка на регистрацию
- Контактное лицо
- Телефон
- Email

### Блок 4. Материалы для публикации

- Черновой текст анонса
- Ссылка на фото/материалы
- Нужна ли афиша

## Валидации

- название обязательно;
- дата и время обязательны;
- площадка обязательна;
- хотя бы один контакт обязателен;
- длина краткого описания не менее 80 символов;
- регистрационная ссылка обязательна, если отмечено, что запись нужна.

## Поведение после отправки

- пользователю показывается сообщение:
  `Спасибо! Событие отправлено в редакцию портала. После проверки оно появится в афише.`
- запись создается в Airtable;
- редактор получает уведомление в Zapier digest.

## Встраивание в Outline

Рекомендуемые страницы:

- `Подать событие`
- `Для координаторов площадок`
- внутренняя страница `Редакция / Intake`
