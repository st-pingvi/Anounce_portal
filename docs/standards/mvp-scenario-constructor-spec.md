# MVP Scenario Constructor Spec

Документ описывает требования к MVP-конструктору сценариев для Репозитория ГОС 2.0.

## Поля ввода

- `topic` — тема сценария из утвержденного справочника.
- `district` — район или площадка.
- `location_profile` — тип пространства и его ограничения.
- `audience` — основная аудитория.
- `audience_segments` — детальные сегменты.
- `duration` — длительность.
- `goal` — целевой результат.
- `resources` — необходимые ресурсы.
- `partners` — партнеры или внешние спикеры.
- `declared_participants` — заявленное количество участников.
- `acquisition_channels` — каналы привлечения.
- `castdev_confirmed` — подтвержден ли запрос от аудитории.
- `feedback_channel` — форма обратной связи или бот.

## Логика генерации

Система должна строить каркас сценария по цели, теме и аудитории.

Примеры:

- цель `traditions` добавляет мастер-классы, просветительский блок и концерт;
- сегмент `nannies_with_grandkids_3_7` добавляет совместную активность;
- тема `digital_literacy` требует уточнения уровня группы;
- тема `business` в ряде случаев переводится из лекции в практикум.

## Правила фильтра

- блокировка несоответствия формата месту;
- проверка на эйджизм;
- запрет метки эксклюзивности для флагманских площадок;
- блокировка показа фильмов без подтвержденной лицензии.

## Экран результата

Результат должен содержать:

- визуальный таймлайн;
- чек-лист ресурсов;
- предупреждения по посещаемости, конфликтности и нагрузке на персонал;
- блок обратной связи.

## Мобильный слой MVP

Интерфейс может быть реализован как мобильное приложение с тремя основными экранами:

- каталог сценариев;
- конструктор;
- экран результата.

Дополнительно рекомендуется поддерживать:

- быстрые фильтры для вдохновения;
- кнопку создания чат-бота для записи;
- ссылку на мини-апп навигации участников.
