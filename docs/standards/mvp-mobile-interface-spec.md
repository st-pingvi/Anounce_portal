# MVP Mobile Interface Spec

Документ фиксирует мобильный интерфейс MVP Репозитория ГОС 2.0.

## Экран 1. Каталог сценариев

Назначение: быстрый поиск и вдохновение.

Обязательные элементы:

- поисковая строка;
- быстрые фильтры-кнопки;
- карточки сценариев с названием, миниатюрой и меткой охвата;
- акцент на проверенные и успешные форматы.

Рекомендуемые быстрые фильтры:

- `top_success`
- `with_grandkids`
- `dacha_season`

## Экран 2. Конструктор

Назначение: быстрая сборка мероприятия без лишней бюрократии.

Обязательные поля:

- `topic`
- `audience`
- `location_profile`
- `resources`
- `castdev_confirmed`

Обязательная логика:

- тренажерный зал ограничивает неподходящие лекционные темы;
- система учитывает наличие спикеров, партнеров и материалов;
- отсутствие кастдева должно блокировать сохранение.

## Экран 3. Результат

Назначение: показать готовый каркас и подсветить риски.

Обязательные элементы:

- таймлайн по блокам;
- список материалов;
- предупреждения;
- действия для запуска записи и навигации.

## Автоматизация

MVP может поддерживать:

- кнопку создания чат-бота для записи;
- ссылку на мини-апп навигации участников.
