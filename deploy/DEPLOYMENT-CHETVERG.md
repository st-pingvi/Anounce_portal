# Размещение портала на chetverg.3321616.ru

Поддомен `chetverg.3321616.ru` уже указывает на IP `85.239.41.111`, но сейчас на этом IP запросы попадают в существующий `nginx`, который редиректит на `/dashboard`.

## Что это значит

На сервере пока нет отдельного virtual host для `chetverg.3321616.ru`, поэтому поддомен не обслуживает портал и уходит в дефолтный сайт.

## Что подготовлено

- `deploy/Caddyfile.chetverg` — конфигурация Caddy для отдельного сайта на поддомене
- `deploy/nginx-chetverg-3321616-portal.conf` — отдельный `server` block для `nginx`
- `prototype-ui/` — статический портал

## Рекомендуемый вариант

Если на сервере уже работает `nginx`, проще всего:

1. Скопировать `prototype-ui/` в `/var/www/gos-portal/`
2. Подключить `deploy/nginx-chetverg-3321616-portal.conf`
3. Выпустить сертификат на `chetverg.3321616.ru`
4. Перезагрузить `nginx`

## Проверка после включения

```bash
curl -I http://chetverg.3321616.ru
curl -I https://chetverg.3321616.ru
```

Ожидаемо:

- HTTP либо сразу отдает сайт, либо редиректит на HTTPS
- HTTPS открывает портал с сертификатом на `chetverg.3321616.ru`

## Почему не получилось разместить автоматически отсюда

В этом окружении нет SSH-доступа к серверу `85.239.41.111`, поэтому я смог подтвердить конфликт маршрутизации, но не могу сам изменить боевой `nginx` или выпустить сертификат на сервере.
