# Размещение портала на 3321616.ru/gos

Подготовлен вариант деплоя для статического портала по пути `/gos`.

## Что уже готово

- `deploy/Caddyfile` — конфигурация Caddy для сайта `3321616.ru` с порталом на `/gos`
- `deploy/nginx-3321616-gos.conf` — шаблон для существующего `nginx`, если dashboard уже работает на этом сервере
- `deploy/docker-compose.yml` — запуск контейнера Caddy
- `prototype-ui/` — статический портал

## Рекомендуемый сценарий

Так как `3321616.ru` уже обслуживает dashboard, безопаснее не поднимать отдельный сайт поверх существующего `nginx`, а добавить новый маршрут:

1. Оставить текущий dashboard на `/dashboard`
2. Разместить файлы портала в отдельной директории, например `/var/www/gos-portal/`
3. Добавить маршрут `/gos` и `/gos/` в существующий `nginx`
4. Перевыпустить сертификат на `3321616.ru`, если TLS уже не покрывает этот хост

## Вариант 1: интеграция в существующий nginx

Используйте шаблон `deploy/nginx-3321616-gos.conf` и встроите его в текущий server block для `3321616.ru`.

Ключевая логика:

- `/gos` -> `308 /gos/`
- `/gos/` отдает статический портал
- `/dashboard` продолжает вести в существующее приложение

## Вариант 2: Caddy для всего домена

```bash
cd /opt/gos20-portal/deploy
docker compose up -d
```

Этот вариант подходит только если Caddy будет фронтовым сервером для всего `3321616.ru`, а не будет конкурировать с уже запущенным `nginx` на тех же портах.

## Проверка

```bash
curl -I http://3321616.ru/gos
curl -I https://3321616.ru/gos
curl -I https://3321616.ru/dashboard
```

Ожидаемо:

- `/gos` перенаправляется на `/gos/`
- `/gos/` открывает портал
- `/dashboard` продолжает открывать существующее приложение

## Что критично проверить

- маршрут `/gos` не попадает в общий `try_files` действующего приложения;
- блок `/gos/` расположен выше catch-all маршрутов;
- `alias` или `root` указывает именно на каталог с файлами портала;
- TLS-сертификат действительно выписан на `3321616.ru`.

## Почему проблема возникала раньше

Сейчас `3321616.ru` уже обслуживает другое приложение и уводит пользователя на `/dashboard`. Значит, без явного маршрута `/gos` запросы не попадут в статический портал.
